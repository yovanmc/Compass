# Compass v3 — Recommender Depth & Evaluation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deepen the pure recommendation engine (MMR diversity, an evaluation harness) and make its quality measurable and demoable without API keys (baked-in sample data), plus surface contribution bars and "more like this" in the UI.

**Architecture:** Keep `Compass.Recommender` pure (MMR, evaluator, similarity = feature-vector ops). Core gains a `WhyFeature` DTO + `Diversity` config + `SimilarTo`. Data gains a `SampleDataProvider` + clear-library. App gains a Diversity slider, contribution bars, a "more like this" section, and Load/Clear sample-data actions. No DB schema change; theme unchanged.

**Tech Stack:** C# / .NET 10, WPF (`net10.0-windows`) + WPF-UI 4.3.0, Microsoft.Data.Sqlite, CommunityToolkit.Mvvm, xUnit + FluentAssertions.

```csharp
using Compass.Recommender;
using FluentAssertions;
using Xunit;

public class MmrDiversityTests
{
    // Helper: a 1.0-weighted ProfileItem / CandidateItem from feature keys.
    static FeatureVector Vec(params string[] keys) => FeatureVector.FromKeys(keys); // adapt to actual factory
    // NOTE: implementer — use the real FeatureVector construction used elsewhere in the suite.

    [Fact]
    public void DiversityZero_ReproducesPureRelevanceOrder()
    {
        var liked = new[] { new ProfileItem("L1", Vec("genre:strategy","theme:sci-fi"), 1.0) };
        var cands = new[]
        {
            new CandidateItem("A", Vec("genre:strategy","theme:sci-fi")),     // very relevant
            new CandidateItem("B", Vec("genre:strategy","theme:sci-fi")),     // very relevant (dup-ish)
            new CandidateItem("C", Vec("genre:strategy")),                    // less
            new CandidateItem("D", Vec("genre:puzzle")),                      // least
        };
        var rec = new ContentRecommender();
        var z = rec.Recommend(liked, cands, new RecommenderOptions { Diversity = 0.0 }, null);
        var pure = z.Recommendations.OrderByDescending(r => r.Score).Select(r => r.ItemId);
        z.Recommendations.Select(r => r.ItemId).Should().Equal(pure);
    }

    [Fact]
    public void DiversityHigh_PromotesTheDistinctCandidate()
    {
        var liked = new[] { new ProfileItem("L1", Vec("genre:strategy","theme:sci-fi"), 1.0) };
        var cands = new[]
        {
            new CandidateItem("A", Vec("genre:strategy","theme:sci-fi")),
            new CandidateItem("B", Vec("genre:strategy","theme:sci-fi")),
            new CandidateItem("C", Vec("genre:strategy","theme:sci-fi")),
            new CandidateItem("D", Vec("genre:strategy","theme:fantasy")),    // distinct theme
        };
        var rec = new ContentRecommender();
        var div = rec.Recommend(liked, cands, new RecommenderOptions { Diversity = 0.6 }, null);
        div.Recommendations.Take(2).Select(r => r.ItemId).Should().Contain("D");
    }
}
```

- [ ] **Step 3: Implement MMR** in `ContentRecommender.Recommend`. After computing the relevance-scored list (each with its ItemId, Score, and the normalized candidate vector available), if `options.Diversity > 0`, replace the final ordering with a greedy MMR selection:

## Phase 2 — RecommenderEvaluator (engine, TDD)

**Outcome:** a pure, reusable evaluator producing recall@k, intra-list diversity, distinct-feature coverage, and score-spread metrics.

### Task 2.1: Evaluator — failing tests

**Files:** Create `tests/Compass.Recommender.Tests/RecommenderEvaluatorTests.cs`.

- [ ] **Step 1: Write failing tests** for each metric on tiny hand-built inputs where the expected value is computable by hand:
  - `RecallAtK`: given 2 liked items, leave-one-out, with the held-out item present as a candidate and obviously most-similar → recall@1 should be 1.0; with k smaller than its rank → lower.
  - `IntraListDiversity`: for a list of two orthogonal vectors → `1 − cosine = 1.0`; for two identical → `0.0`.
  - `DistinctFeatureCoverage`: union of feature keys across the top-N has the expected count.
  - `ScoreSpread`: min/max/mean for a known score list.

```csharp
using Compass.Recommender;
using FluentAssertions;
using Xunit;

public class RecommenderEvaluatorTests
{
    [Fact]
    public void IntraListDiversity_OrthogonalIsOne_IdenticalIsZero()
    {
        var eval = new RecommenderEvaluator();
        var a = FeatureVector.FromKeys("genre:strategy");
        var b = FeatureVector.FromKeys("genre:puzzle");
        eval.IntraListDiversity(new[] { a, b }).Should().BeApproximately(1.0, 1e-9);
        eval.IntraListDiversity(new[] { a, a }).Should().BeApproximately(0.0, 1e-9);
    }

    [Fact]
    public void RecallAtK_HeldOutMostSimilar_RanksFirst()
    {
        var eval = new RecommenderEvaluator();
        var liked = new[]
        {
            new ProfileItem("X", FeatureVector.FromKeys("genre:strategy","theme:sci-fi"), 1.0),
            new ProfileItem("Y", FeatureVector.FromKeys("genre:strategy","theme:sci-fi"), 1.0),
        };
        // Candidates include the held-out's twin so it should rank top.
        var recall = eval.LeaveOneOutRecallAtK(liked, k: 1, new RecommenderOptions());
        recall.Should().BeGreaterThan(0.0);
    }
}
```
